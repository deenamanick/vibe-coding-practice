# Module 13: Deployment Using Free Platforms

This module teaches you how to take your application from your local computer and launch it on the internet for the world to see, using completely free tools.

## Practicals

- `module-13-1-git-github-basics.md` — Git & GitHub (Saving your progress in the cloud)
- `module-13-2-vercel-frontend-deploy.md` — Vercel (Fast & free frontend deployment)
- `module-13-3-railway-backend-deploy.md` — Railway (Deploying your Node.js/Python server)
- `module-13-4-supabase-database-deploy.md` — Supabase (Cloud database setup)
- `module-13-5-production-env-config.md` — Production Config (Managing secrets in the cloud)

### Recommended order

1. Git & GitHub Basics
2. Vercel (Frontend)
3. Railway (Backend)
4. Supabase (Database)
5. Production Env Config

### Prerequisites

- A GitHub account ([github.com](https://github.com))
- Completed Module 5 (Express/FastAPI)
- Completed Module 11 (Security/.env)

### 🌍 Why These Platforms?

**We chose these because they are student-friendly:**
- ✅ **Vercel**: Best for frontend, automatically deploys from GitHub.
- ✅ **Railway**: Great for backend, has a generous free "Trial" tier.
- ✅ **Supabase**: The best free database for students (PostgreSQL).
- ✅ **No Credit Card Required**: These options typically don't require a card for basic usage.

### Tech Stack

- **Version Control**: Git, GitHub
- **Frontend Hosting**: Vercel
- **Backend Hosting**: Railway
- **Database**: Supabase (PostgreSQL)
- **Configuration**: Environment Variables in Production
